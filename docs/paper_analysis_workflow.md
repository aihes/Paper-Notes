# 通用内容分析与博客撰写工作流 (SOP)

本文档旨在固化一套标准的、高质量的技术文章/论文阅读、分析及博客撰写工作流程。所有未来处理类似分析任务时，均应遵循此指南。

---

## 阶段一：源内容获取与初始化

1.  **创建专属目录**：
    -   根据文章/论文的主题，在 `papers/` 目录下创建一个新的、语义化的子目录。目录名应使用小写字母和连字符（kebab-case），例如 `papers/continuous-autoregressive-language-models`。

2.  **获取原始内容**：
    -   **情况 A：如果源是 PDF 文件**
        -   使用 `curl` 将 PDF 下载到新创建的目录中。
        -   使用 `scripts/read_paper.py` 脚本，将 PDF 的文本内容提取出来，并保存为 `source_content.md`。
        -   **注意**：如遇依赖缺失（如 `PyMuPDF`），应先安装。
    -   **情况 B：如果源是 HTML 网页/文章**
        -   使用 `scripts/html_to_md.py` 脚本，将网页内容抓取并转换为 Markdown 格式，保存为 `source_content.md`。
        -   **注意**：该脚本可能需要特定依赖（如 `beautifulsoup4`, `markdownify`），按需安装。

3.  **创建基础 README**：
    -   在目录下创建一个 `README.md` 文件，并将文章/论文的标题作为一级标题写入，为后续笔记和任务管理做准备。

## 阶段二：核心内容撰写

1.  **深度阅读与分析**：
    -   仔细阅读 `source_content.md` 的内容，完全消化文章/论文的核心思想、技术细节和结论。

2.  **撰写分析文章初稿**：
    -   直接在当前目录的 `README.md` 文件中撰写分析文章。该文件在阶段一已创建。
    -   分析文章应结构清晰、逻辑严谨。初稿应至少包含以下部分：
        -   **问题陈述**：文章/论文试图解决的核心问题是什么？
        -   **核心思想/方案**：提出的主要解决方案或范式是什么？
        -   **关键技术细节**：分点阐述方案中的关键技术或实现细节。
        -   **结果/结论**：文章/论文通过哪些实验或论证得出了什么结论？
        -   **总结与展望**：总结其贡献，并提及未来可行的方向。

## 阶段三：丰富化与润色（Checklist）

根据以下清单，对 `README.md` 进行全面优化和丰富。这是一个强制检查列表，每一项都必须完成。

-   [ ] **1. 添加“一句话总结 (TL;DR)”**：
    -   在文章最顶部，用一个简短、精炼的段落概括最核心的贡献和结论。

-   [ ] **2. 添加原文链接**：
    -   在文章的最顶部，用引用的方式明确指出本文是对哪篇文章/论文的解读，并附上原文链接。

-   [ ] **3. 添加并验证图片**：
    -   在分析文章中引用源内容中最关键的图表（Figures/Images）。
    -   在当前目录下创建一个 `images/` 子目录用于存放图片。
    -   请求用户提供这些图片，并确认其文件名。
    -   **务必**根据用户提供的实际文件名，更新 `README.md` 中的图片引用路径，确保图片能正确显示。

-   [ ] **4. 添加“核心术语解释 (Glossary)”**：
    -   在文章末尾，创建一个“附录”或“核心术语解释”部分。
    -   挑选文章中对于非专家读者可能比较晦涩的关键技术术语，并用通俗易懂的语言进行解释。

-   [ ] **5. 添加“参考文献 (References)”**：
    -   在术语解释之后，添加一个参考文献部分。
    -   至少应包含对本文所分析的原始文章/论文的正式引用。

## 阶段四：最终审查

1.  **完整性检查**：对照**阶段三的 Checklist**，确保所有要求均已满足。
2.  **格式与可读性**：通读全文，检查排版、措辞和逻辑流畅性，确保文章专业且易于阅读。
    -   **语言风格**：确保语言客观、中立，避免使用夸张、主观性强的形容词和比喻（例如，避免使用“革命性”、“浪潮之巅”等词汇），保持技术文章的严谨性。
3.  **提交最终成品**：在确认所有步骤完成后，提交当前目录下内容完善的 `README.md` 作为任务成果。
